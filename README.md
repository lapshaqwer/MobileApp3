## Контрольные вопросы:
1. ViewBinding - инструмент для безопасного доступа к View без findViewById, подключается в build.gradle (buildFeatures { viewBinding true }) и используется через сгенерированный класс.
2. Декларативная подписка (android:onClick в XML) проще для статических экранов, но требует точной сигнатуры метода (View param), иначе NoSuchMethodException; программная (setOnClickListener) гибче для динамической логики и переиспользования.
3. Приложение упадет с NoSuchMethodException, потому что Android через рефлексию ищет метод с точной сигнатурой public void имя(View v), а переданный параметр View необходим для идентификации нажатого компонента.
4. Жизненный цикл Activity: onCreate - onStart - onResume - onPause - onStop - onDestroy; слушатели лучше инициализировать в onCreate, т.к. View уже созданы и метод вызывается один раз.
5. Анонимный внутренний класс - безымянный класс, создаваемый на лету для одноразовой реализации интерфейса, чаще всего используется для слушателей событий (setOnClickListener(new View.OnClickListener() {...})).
